Model API¶

The models submodule contains a basic general model API class which can easiliy be implemented and, if required extended, for individual models. The general model API contains methods to read, write and build models and ensures a common interface to different models. Workflows can be combined and wrapped into methods such as the build method of the model class.

HydroMT has a general model class which has identical methods for each model. The most important methods are discussed here and model specific methods in each of subsections. A full list of all methods can be found in the reference API.

Model API:

  • Properties:

    • root: path to model root.

    • config: dict() interface to the model configuration.

    • staticmaps: an xarray.Dataset() interface to all model staticmaps.

    • staticgeoms: a dict() of geopandas.GeoDataFrame() interface to all model geometry files.

    • crs: the model coordinate reference system.

    • bounds: the model bounding box.

    • region: the geomtery of model bounding box as geopandas.GeoDataFrame().

  • Methods:

    • build(): method to build a complete model from scratch.

    • setup_component(): method to update or setup a component once the basemaps have been setup.

    • read_config() / write_config(): parse model config to and from file.

    • get_config(): get a config option

    • set_config(): set a value to a config option

    • read_staticmaps() / write_staticmaps(): parse model maps to and from disk (often several files in model specific format).

    • set_staticmaps(): add a map to the model staticmaps.

    • read_staticgeoms() / write_staticgeoms(): parse model geometry data to and from file.

    • set_staticgeoms(): Add a geometry to the model staticgeoms.
